## 方法一：通过 Cloudflare Dashboard 部署（推荐新手）

### 第一步：准备 GitHub 仓库

1. 在 [github.com](https://github.com) 注册并登录
2. 点击右上角 `+` → `New repository`
3. 仓库名填 `math-pro`，选 `Private`，点 `Create repository`
4. 把本部署包所有文件按照目录结构上传到仓库：
   - 点 `Add file` → `Upload files`
   - 上传时**保持目录结构**（`public/` 和 `functions/` 两个文件夹）

### 第二步：连接 Cloudflare Pages

1. 登录 [dash.cloudflare.com](https://dash.cloudflare.com)
2. 左侧菜单选 **Workers & Pages**
3. 点击 **Create application** → **Pages** → **Connect to Git**
4. 授权 GitHub，选择刚才创建的 `math-pro` 仓库
5. 构建配置如下：

   | 设置项 | 值 |
   |--------|-----|
   | 项目名称 | `math-pro`（或自定义） |
   | 生产分支 | `main` |
   | 框架预设 | 无 / None |
   | 构建命令 | （留空） |
   | 构建输出目录 | `public` |

6. 点击 **Save and Deploy**，等待 1-2 分钟完成首次部署

### 第三步：配置 DeepSeek API 密钥（关键）

1. 部署完成后，点击项目名称进入项目页面
2. 点击顶部 **Settings** 选项卡
3. 左侧选 **Environment variables**
4. 点击 **Add variable**，填写：
   - 变量名：`DEEPSEEK_API_KEY`
   - 值：你的 DeepSeek API Key（从 [platform.deepseek.com](https://platform.deepseek.com) 获取）
   - 选择 **Production** 和 **Preview** 都勾选
   - 类型选 **Secret**（加密存储）
5. 点击 **Save**

6. **重新部署**：回到 Deployments 页面，点击最新部署右侧的 `...` → **Retry deployment**

### 第四步：访问你的应用

部署成功后，Cloudflare 会给你一个域名，格式为：

```
https://math-pro-xxx.pages.dev
```

在浏览器打开即可使用。

## 方法二：通过 Wrangler CLI 部署（适合开发者）

### 安装 Wrangler

```bash
npm install -g wrangler
```

### 登录 Cloudflare

```bash
wrangler login
```

### 部署

```bash
cd math-pro
wrangler pages deploy public --project-name=math-pro
```

### 设置环境变量

```bash
wrangler pages secret put DEEPSEEK_API_KEY --project-name=math-pro
# 输入你的 DeepSeek API Key，回车确认
```

## 获取 DeepSeek API Key

1. 访问 [platform.deepseek.com](https://platform.deepseek.com)
2. 注册账号并登录
3. 点击左侧 **API Keys**
4. 点击 **Create new secret key**
5. 复制 Key（只显示一次，请妥善保存）
6. DeepSeek 新用户有免费额度，正常家庭使用费用极低

## 常见问题排查

### 问题：点击「生成题目」没有反应或报错

**检查：**

1. 确认环境变量 `DEEPSEEK_API_KEY` 已正确设置
2. 确认已重新部署（修改环境变量后必须重新部署）
3. 打开浏览器 F12 → Console，查看错误信息
4. 检查 DeepSeek 账户余额是否充足

### 问题：Functions 不工作（API 404）

**检查：**

- 确认 `functions/api/chat.js` 文件存在于仓库根目录的 `functions/api/` 路径下
- 构建输出目录必须填 `public`（不是根目录）
- Cloudflare Pages Functions 要求文件放在项目根目录的 `functions/` 下，而静态文件放在 `public/` 下

### 问题：页面可以访问但语音识别不工作

浏览器语音识别需要 HTTPS，Cloudflare Pages 默认提供 HTTPS，应该正常工作。
如果还有问题，确保使用 Chrome 浏览器。

### 问题：页面中文显示乱码

确认 `public/index.html` 文件以 **UTF-8** 编码保存，且包含 `<meta charset="UTF-8">`。本项目已按 UTF-8 配置，正常部署不应出现乱码。

## 数据存储说明

所有练习数据（练习记录、闯关记录、孩子档案、题库等）存储在**浏览器本地 localStorage** 中，不会上传到任何服务器，保护隐私。

注意：清除浏览器缓存会导致数据丢失，建议定期导出（未来版本将支持）。
